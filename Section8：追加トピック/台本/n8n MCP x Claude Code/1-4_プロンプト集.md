# プロンプト集 — n8n MCP　自然言語でワークフローを自動生成しよう

## 1. 接続確認

```
n8nのMCPで使えるツールを一覧してください。
```

## 2. 設計を相談する（まだ作らせない）

```
n8nで業務自動化のワークフローを作りたいです。
Webの問い合わせフォームに入力が来たら、その内容をGoogleスプレッドシートに1行追加して、
Slackの #n8n-test チャンネルに「新しい問い合わせがありました」と通知するワークフローを作りたい。

記録用のスプレッドシートは、私のGoogleドライブにある「AI+テスト」という既存フォルダの中に、
このワークフロー構築と一緒に新しく作ってください（手作業で用意するのは省きたい）。

まだ作らないでください。まず、
1. どのノードを使うか（スプレッドシートの新規作成もどのノードで行うか含めて）
2. それぞれのノードで何を設定する必要があるか
3. 事前に私が手作業で用意しておくべきもの（認証など）
を教えてください。
```

## 3. ワークフローを作らせる（検証込み・オフで作成）

```
その構成でワークフローを作ってください。
- 記録用スプレッドシートは「AI+テスト」フォルダ内に新規作成
- 1行目に見出し「日時 / 名前 / メール / 問い合わせ内容」を入れ、実務でそのまま使えるよう見やすく整えてください
  （見出し行は分かりやすい日本語で。可能なら見出し行の固定や太字など、体裁も整える）
- フォームの入力項目は「名前」「メールアドレス」「問い合わせ内容」の3つ
- スプレッドシートには「日時 / 名前 / メール / 問い合わせ内容」を記録
- Slack通知には送信者の名前と問い合わせ内容を含める
作成したら、問題がないか検証してから報告してください。
本番ではなくテスト用として、オフ（非アクティブ）の状態で作ってください。
```

## 4. テスト実行させる

```
このワークフローをテスト実行して、結果を教えてください。
```

## 5. 不具合を調べて直してもらう（デモ用ワークフロー）

```
n8nに「営業メール返信→ステータス記録（デモ用）」というワークフローがあります。
実行すると、スプレッドシートの「次アクション」列だけ毎回空になります。原因を調べて、修正してください。
また、修正前と修正後で何が変わったのかを、n8nを知らない人にも分かるように説明してください。
```

## 6. 自分のワークフローで詰まったとき（汎用）

```
実行したら〔ここに実際のエラーメッセージや症状を貼る〕という状態になりました。
原因を特定して、ワークフローを修正してください。
また、修正前と修正後で何が変わったのかを、n8nを知らない人にも分かるように説明してください。
```
