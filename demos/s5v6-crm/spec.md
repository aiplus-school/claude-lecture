# 顧客管理アプリ — 仕様書

## 目的

Excelで顧客台帳を管理する代わりに、ブラウザだけで顧客と商談を登録・検索・編集でき、見込み→提案→成約のパイプラインを3列のカンバンで見渡せる1人営業用アプリ。
業務用CRM SaaSの核機能（顧客・商談・パイプライン）を最小構成で再現し、仕組みを理解することが目的（メール送信連携・カレンダー連携などは対象外）。

## 利用シーン

- 展示会・打ち合わせで知り合った顧客の連絡先と背景メモを残す
- 顧客ごとに進行中の商談を紐付け、金額・ステータス・フォローアップ内容を記録する
- 商談を「見込み」「提案」「成約」の3列カンバンで眺め、どこに案件が滞留しているか把握する
- ステータスを進めたい商談を見つけたら、その場で次のステータスへ動かす

## データ項目

### 顧客（1件）

| 項目 | 必須 | 備考 |
|---|---|---|
| id | ○ | アプリが自動採番 |
| 会社名 | ○ | 例: 株式会社サンプル |
| 担当者名 | ○ | 例: 山田 太郎 |
| 役職 | | 任意 |
| メール | | 任意 |
| 電話 | | 任意 |
| メモ | | 用途・背景を自由記述、複数行可 |
| 作成日時 | ○ | アプリが自動付与（ISO文字列） |

### 商談（1件）

| 項目 | 必須 | 備考 |
|---|---|---|
| id | ○ | アプリが自動採番 |
| customerId | ○ | 顧客のidと一致（1対多の親） |
| タイトル | ○ | 例: サービスA導入提案 |
| 金額 | | 整数・円（任意） |
| ステータス | ○ | 見込み / 提案 / 成約 のいずれか。初期値は「見込み」 |
| フォローアップメモ | | 任意、複数行可 |
| 作成日時 | ○ | アプリが自動付与 |
| 更新日時 | ○ | 保存のたびに更新 |

**1対多の関係**: 1人の顧客に対して0〜複数の商談がぶら下がる。商談は `customerId` で顧客を参照する。

## 画面構成

画面上部に2つのビュー切替タブを置く。タブをクリックすると下のメインエリアの中身が丸ごと切り替わる。

### ビュー① 顧客（ツーカラム）

左右2ペイン構成。左は常時一覧、右は選択・操作に応じて中身を切り替える。

#### 左ペイン（幅340px 固定）

- 上部: 検索ボックス（会社名・担当者名・役職を横断検索）と「＋ 新規顧客」ボタン
- 下部: 顧客カードの縦リスト（作成日時の新しい順、スクロール可）
- 各カード: 会社名／担当者名／役職／紐付く商談の件数
- 選択中のカードはハイライトする（左にオレンジの縦ライン＋薄いオレンジ背景）
- カードをクリックで右ペインに顧客詳細を表示

#### 右ペイン（モード切替: 空 / 顧客詳細 / 顧客フォーム / 商談フォーム）

**モード1: 空状態**
- 何も選んでいない初期状態
- 「左から顧客を選んでください」のガイドを中央に表示

**モード2: 顧客詳細**
- 上部: 右側に「編集」「削除」ボタン
- 連絡先ブロック: 会社名（大見出し）／担当者名／役職／メール／電話／メモ
- 商談一覧ブロック: 紐付く商談の行リスト＋「＋ 商談を追加」ボタン
- 商談行: タイトル／金額／ステータスバッジを1行で表示。クリックで商談フォーム（編集）に遷移
- 削除時は確認ダイアログを出す。紐付く商談も連鎖削除されることを明示する

**モード3: 顧客フォーム（新規・編集）**
- 上部: 「キャンセル」ボタン／タイトル（新規顧客 or 編集）
- 中央: 各項目の入力フィールド（会社名・担当者名・役職・メール・電話・メモ）
- 下部: 「保存」ボタン
- 保存後は顧客詳細モードへ切り替わる

**モード4: 商談フォーム（新規・編集）**
- 上部: 「キャンセル」ボタン／タイトル（新規商談 or 商談を編集）／親となる顧客の会社名を小さく表示
- 中央: タイトル・金額・ステータス（セレクトボックス）・フォローアップメモ
- 下部: 「保存」ボタン（編集時は「削除」ボタンも表示）
- 保存・削除後は顧客詳細モードへ戻る

### ビュー② パイプライン（カンバン3列）

- 3列を左から：「見込み」「提案」「成約」
- 各列のヘッダ: ステータス名と件数
- 各列に該当ステータスの商談カードを並べる
- カード本文: 商談タイトル／紐付く顧客の会社名／金額
- カード左に3pxのアクセントライン（ステータス色）
- カード右下に「←」「→」ボタン。クリックで隣のステータスに移動（先頭列では「←」、末尾列では「→」を非表示）
- カードをクリックすると「顧客」ビューに戻り、該当顧客の商談フォーム（編集モード）を開く

## 操作フロー（CRUD）

### 顧客

| 操作 | 遷移 |
|---|---|
| 作成 (Create) | 顧客ビュー → ＋新規顧客 → 顧客フォーム → 保存 → 顧客詳細 |
| 一覧 (Read) | 起動時に左ペインで表示。検索ボックスで絞り込み |
| 詳細 (Read) | カードクリック → 顧客詳細 |
| 更新 (Update) | 顧客詳細 → 編集 → 顧客フォーム（既存値入り）→ 保存 → 顧客詳細 |
| 削除 (Delete) | 顧客詳細 → 削除 → 確認 → 紐付く商談も連鎖削除 → 空状態 |

### 商談

| 操作 | 遷移 |
|---|---|
| 作成 (Create) | 顧客詳細 → ＋商談を追加 → 商談フォーム → 保存 → 顧客詳細 |
| 一覧 (Read) | 顧客詳細の商談一覧 or パイプラインビュー |
| 詳細 (Read) | 商談行クリック → 商談フォーム（編集） |
| 更新 (Update) | 商談フォーム → 保存 → 顧客詳細 |
| 削除 (Delete) | 商談フォーム → 削除 → 確認 → 顧客詳細 |
| ステータス遷移 | パイプラインビューの「←」「→」で隣の列へ移動（フォームを開かず直接更新） |

## 永続化

- 顧客は `localStorage` のキー `crm-customers` に配列として保存
- 商談は `localStorage` のキー `crm-deals` に配列として保存
- 起動時に両方を読み込み、変更時に都度書き込む
- 商談は `customerId` で顧客を参照する
- 顧客削除時は、同じ `customerId` を持つ商談をまとめて削除する（連鎖削除）

## 初期データ

空だと触感がゼロなので、初回起動時（localStorageが空のときのみ）にサンプル顧客3件・商談5件を投入する。
- 会社名・人物名は実在しないものを使う
- 商談は「見込み」「提案」「成約」の3ステータスに分散させる
- 金額もバラけさせる

## 対象外（今回作らない）

- ドラッグ＆ドロップでのステータス移動（クリックの「←」「→」で代替）
- クラウド保存・他端末共有（S6でSupabase連携するときに拡張）
- ログイン・マルチユーザー
- インポート／エクスポート（CSV等）
- メール送信連携・カレンダー連携
- 売上集計・予測レポート
- モバイル対応（PC表示のみ）

## デザイン方針

- 顧客ビューはツーカラムレイアウト（左=顧客リスト、右=詳細・フォーム）
- パイプラインビューは3列カンバンレイアウト
- 白基調、アクセントはClaudeオレンジ `#c15f3c`
- 余白多め、角丸 `rounded-lg`、影は控えめ
- ステータスバッジで視覚的に状態を区別（見込み=グレー、提案=オレンジ、成約=グリーン）
- モバイル対応は今回スコープ外（PC表示を前提とする）
