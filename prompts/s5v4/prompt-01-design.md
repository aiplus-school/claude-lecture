これから名刺管理アプリを作ります。実装に入る前に、CLAUDE.md と spec.md の2枚のドキュメントを書いてください。

---

## 1. 作りたいアプリ

ひとり用の名刺管理アプリ。紙の名刺を取っておく代わりに、ブラウザ1枚で登録・一覧・検索・編集・削除ができる最小構成。業務用SaaSの核機能（OCRを除くCRUD）を再現する。

---

## 2. 技術制約

- HTML / JavaScript（vanilla、フレームワークなし）
- Tailwind CSS は CDN で読み込む
- データは localStorage に保存（サーバー・DB・外部API は使わない）
- ビルドツール・npm は使わない
- ファイルは index.html / app.js / styles.css の3枚で完結させる

---

## 3. 画面構成

### 3.1 全体レイアウト

ツーカラム構成。左ペインで一覧・検索、右ペインで詳細・フォームを扱う。モバイル対応は対象外（PC表示のみ）。

### 3.2 左ペイン（幅 340px 固定）

- 上部に検索ボックスと「＋ 新規登録」ボタン
- 下部に名刺カードの縦リスト
- 選択中のカードはハイライト表示

### 3.3 右ペイン（flex-1、モード切替）

以下の3モードを切り替えて表示する。

- 空状態（何も選んでいない初期表示）
- 詳細（選択中の名刺を表示）
- 登録・編集フォーム

---

## 4. データ項目（名刺1件）

| 項目 | キー | 必須 | 備考 |
|---|---|---|---|
| ID | id | ○ | 自動採番 |
| 氏名 | name | ○ | 例: 山田 太郎 |
| 会社名 | company | ○ | 例: 株式会社サンプル |
| 役職 | title |  | 任意 |
| メール | email |  | 任意 |
| 電話 | phone |  | 任意 |
| メモ | memo |  | 複数行可 |
| 作成日時 | createdAt | ○ | ISO文字列で自動付与 |

localStorage キー: `meishi-cards`（配列で保存）

---

## 5. CRUD操作フロー

- **Create**: 新規登録ボタン → フォーム → 保存 → 詳細表示
- **Read（一覧）**: 起動時に左ペインで一覧表示
- **Read（検索）**: 検索ボックスで氏名・会社名・役職を横断検索
- **Update**: 詳細 → 編集ボタン → フォーム（既存値入り）→ 保存 → 詳細
- **Delete**: 詳細 → 削除ボタン → 確認ダイアログ → 空状態に戻る

---

## 6. デザイン方針

- 白基調、アクセントカラーは `#c15f3c`（Claudeオレンジ）
- フォントは游ゴシック系（`"游ゴシック", "Yu Gothic", sans-serif`）
- 角丸は `rounded-lg`（8px）で統一
- 影は控えめ（`shadow-sm` 程度）
- 並び順は作成日時の新しい順

---

## 7. 初期データ

初回起動時（localStorage が空）のときだけ、サンプル3件を自動投入する。実在しない人物名・会社名を使う。

---

## 8. 対象外（今回作らない）

- OCR（画像からの自動入力）
- クラウド保存・他端末共有
- ログイン・マルチユーザー
- インポート／エクスポート
- モバイル対応
