# spec.md — 名刺管理アプリ 仕様書

## 1. 目的・利用シーン

### 目的
紙で溜まっていく名刺をブラウザ1枚で管理できるようにする、**ひとり用**の最小構成の名刺管理アプリ。業務用SaaSの核機能（OCRを除くCRUD）を再現することを学習ゴールとする。

### 想定利用シーン
- 展示会・商談で受け取った名刺を帰社後に入力し、紙を処分する
- 過去に名刺交換した相手を「会社名」「氏名」「役職」から検索して連絡先を引き出す
- メモ欄に「どこで会ったか」「何を話したか」を残し、次の接点で思い出せるようにする

### 想定ユーザー
- 個人事業主・小規模チームの営業担当
- PCで作業する時間が長い（モバイル対応は不要）
- クラウド同期より、手元で完結する気軽さを優先する

## 2. データ項目

| 項目 | キー | 必須 | 型 | 備考 |
|---|---|---|---|---|
| ID | `id` | ○ | string | `Date.now()` ベースで自動採番 |
| 氏名 | `name` | ○ | string | 例: 山田 太郎 |
| 会社名 | `company` | ○ | string | 例: 株式会社サンプル |
| 役職 | `title` | | string | 任意。例: 営業部長 |
| メール | `email` | | string | 任意。形式バリデーションは簡易 |
| 電話 | `phone` | | string | 任意。書式は自由 |
| メモ | `memo` | | string | 任意。複数行可（`textarea`） |
| 作成日時 | `createdAt` | ○ | string | ISO文字列で自動付与 |

## 3. 画面構成

### 全体レイアウト
ツーカラム構成。左ペインで一覧・検索、右ペインで詳細・フォームを扱う。

```
┌──────────────────────────────────────────────────┐
│  左ペイン（340px固定）  │  右ペイン（flex-1）    │
│  ┌──────────────┐      │                        │
│  │ 🔍 検索ボックス │      │   3モードを切替表示     │
│  └──────────────┘      │   ・空状態             │
│  [＋ 新規登録]           │   ・詳細               │
│  ──────────────         │   ・登録／編集フォーム  │
│  [ 名刺カード1 ]←選択中  │                        │
│  [ 名刺カード2 ]         │                        │
│  [ 名刺カード3 ]         │                        │
│  ...                    │                        │
└──────────────────────────────────────────────────┘
```

### 左ペイン（一覧）
- 上部に検索ボックスと「新規登録」ボタン
- 下部に名刺カードの縦リスト
- 各カードには **氏名・会社名・役職** を表示
- 選択中のカードはオレンジ縦ライン＋薄いオレンジ背景でハイライト
- 検索ボックスに入力があると、**氏名・会社名・役職** を横断検索してリアルタイム絞り込み

### 右ペイン（3モード）

#### 3-1. 空状態（`#pane-empty`）
- 初回起動時・名刺未選択時・削除直後に表示
- 「左から名刺を選択するか、新規登録してください」的な案内文
- 中央寄せ・グレー文字

#### 3-2. 詳細（`#pane-detail`）
- 選択中の名刺を全項目表示
- 右上に「編集」「削除」ボタン
- 「削除」は確認ダイアログ（`confirm`）を経由

#### 3-3. 登録・編集フォーム（`#pane-form`）
- 新規登録・編集の両方で同じフォームを使い回す
- 編集時は既存値を入力欄に復元する
- 氏名・会社名は必須（空欄時は保存ボタンで弾く）
- 「保存」「キャンセル」ボタンを下部に配置

## 4. CRUD操作フロー

| 操作 | トリガー | 遷移 |
|---|---|---|
| **Create** | 左ペインの「＋ 新規登録」ボタン | 右を `pane-form` に切替（空のフォーム）→ 保存 → `pane-detail`（新規カードを選択状態に） |
| **Read（一覧）** | 起動時 / 保存・削除の後 | 左ペインに作成日時の新しい順で全件表示 |
| **Read（検索）** | 検索ボックスへの入力 | 氏名・会社名・役職を横断検索し、左ペインの一覧をリアルタイム絞り込み |
| **Read（詳細）** | 左ペインのカードをクリック | 右を `pane-detail` に切替、選択ハイライトを更新 |
| **Update** | 詳細画面の「編集」ボタン | 右を `pane-form` に切替（既存値入り）→ 保存 → `pane-detail` に戻る |
| **Delete** | 詳細画面の「削除」ボタン | `confirm` で確認 → localStorage から除去 → 左ペイン再描画 → 右は `pane-empty` に戻る |

## 5. 永続化方針

- **ストレージ**: ブラウザの `localStorage`
- **キー**: `meishi-cards`
- **保存形式**: 名刺オブジェクトの配列を JSON 文字列化
- **読み込みタイミング**: アプリ起動時に1回
- **保存タイミング**: Create / Update / Delete の直後に、配列全体を上書き保存
- **排他制御**: 不要（ひとり用・単一タブ想定）

## 7. 対象外の機能

以下は今回のスコープから外す。将来拡張として記録しておくが、今回の実装では一切手を出さない。

- **OCR**（画像から氏名・会社名を自動抽出する機能）
- **クラウド保存・他端末共有**（複数端末間の同期）
- **ログイン・マルチユーザー**（認証・権限管理）
- **インポート／エクスポート**（CSV・vCard・連絡先アプリ連携）
- **モバイル対応**（スマホ用レイアウト・タッチ最適化）
- **タグ・グループ分類**（ラベリング機能）
- **画像添付**（名刺画像の保存）
- **通知・リマインダー**（フォローアップ機能）
