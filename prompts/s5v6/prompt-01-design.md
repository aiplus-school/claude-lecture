## 作りたいアプリ

ひとり営業用の顧客管理アプリ。Excelで顧客台帳を管理する代わりに、ブラウザ1枚で顧客と商談を登録・一覧・編集・削除でき、見込み→提案→成約のパイプラインを3列のカンバンで見渡せる最小構成。業務用CRM SaaSの核機能を再現する。

## 技術制約

- HTML / JavaScript（vanilla、フレームワークなし）
- Tailwind CSS は CDN で読み込む
- データは localStorage に保存（サーバー・DB・外部API は使わない）
- ビルドツール・npm は使わない
- ファイルは index.html / app.js / styles.css の3枚で完結させる

## 画面構成

上部に2つのビュー切替タブ：「顧客」「パイプライン」。

### ビュー① 顧客（ツーカラム）

- 左ペイン（幅340px 固定）: 検索ボックス＋「新規顧客」ボタン＋顧客カードの縦リスト
- 右ペイン（flex-1）: モードを4種で切り替える
    - 空状態（何も選んでいない初期表示）
    - 顧客詳細（連絡先ブロック＋商談一覧＋「商談を追加」ボタン）
    - 顧客フォーム（新規・編集）
    - 商談フォーム（新規・編集）
- 選択中のカードはハイライト表示する
- 商談一覧の各行はステータスバッジ付き。クリックで商談フォーム（編集）に遷移
- モバイル対応は対象外（PC表示のみ）

### ビュー② パイプライン（カンバン3列）

- 3列を左から：「見込み」「提案」「成約」
- 各列に該当ステータスの商談カードを並べる
- カード本文: 商談タイトル＋紐付く顧客の会社名＋金額
- カードの右下に「←」「→」ボタンを置き、クリックで隣のステータスに移動（先頭列では「←」、末尾列では「→」を非表示）
- カードをクリックすると「顧客」ビューに戻り、該当顧客の商談フォーム（編集）を開く

## CRUD操作フロー

### 顧客

- **Create**: 新規顧客ボタン → 顧客フォーム → 保存 → 顧客詳細
- **Read**: 起動時に左ペインで一覧表示／検索ボックスで会社名・担当者名・役職を横断検索
- **Update**: 顧客詳細 → 編集ボタン → 顧客フォーム（既存値入り）→ 保存 → 顧客詳細
- **Delete**: 顧客詳細 → 削除ボタン → 確認ダイアログ → 紐付く商談も連鎖削除 → 空状態に戻る

### 商談

- **Create**: 顧客詳細で「商談を追加」→ 商談フォーム → 保存 → 顧客詳細（商談一覧に反映）
- **Read**: 顧客詳細の商談一覧 or パイプラインビューで表示
- **Update**: 商談クリック → 商談フォーム → 保存
- **Delete**: 商談フォーム内の削除ボタン → 確認ダイアログ → 顧客詳細に戻る
- **ステータス遷移**: パイプラインビューの「←」「→」でも商談の保存を伴わず直接更新可能

## デザイン方針

- 白基調、アクセントカラーは `#c15f3c`（Claudeオレンジ）
- フォントは游ゴシック系（`"游ゴシック", "Yu Gothic", sans-serif`）
- 角丸は `rounded-lg`（8px）で統一
- 影は控えめ（`shadow-sm` 程度）
- ステータスバッジの色: 見込み=グレー、提案=オレンジ、成約=緑
- 顧客の並び順は作成日時の新しい順
- 商談カードはステータス別にカード左に3pxのアクセントライン（同色）

## 初期データ

初回起動時（localStorageが空）のときだけ、顧客3件・商談5件を自動投入する。実在しない会社名・人物名を使う。商談は3つのステータスに分散させる。

## 対象外（今回作らない）

- ドラッグ＆ドロップでのステータス移動（クリックで代替）
- クラウド保存・他端末共有
- ログイン・マルチユーザー
- インポート／エクスポート（CSV等）
- モバイル対応
- メール送信連携・カレンダー連携
