# 経費管理アプリ — 仕様書

## 目的

紙のレシートや表計算ソフトで経費を管理する代わりに、ブラウザだけで経費を登録・検索・精算管理できる1人用アプリ。
月額SaaSの経費管理機能を最小構成で再現し、仕組みを理解することが目的（OCRや承認ワークフローは対象外）。

## 利用シーン

- 出張・打ち合わせで発生した交通費や会議費を都度記録する
- 月末にその月の経費を見返し、まとめて精算済みに切り替える
- カテゴリや日付で過去の経費を検索する

## データ項目（経費1件）

| 項目 | 必須 | 備考 |
|---|---|---|
| id | ○ | アプリが自動採番 |
| 日付 | ○ | 経費が発生した日（YYYY-MM-DD） |
| カテゴリ | ○ | 交通費／会議費／接待費／消耗品／通信費／その他 |
| 金額 | ○ | 円（整数・0円は不可） |
| メモ | | 用途や相手先などを自由記述 |
| ステータス | ○ | 申請中 / 精算済 のいずれか。初期値は「申請中」 |
| 作成日時 | ○ | アプリが自動付与 |

## 画面構成（ツーカラム）

画面は**左右2ペイン**で構成する。左は常時一覧、右は選択・操作に応じて内容を切り替える。

### 左ペイン（常時表示）

- 上部: 検索ボックス（メモ・カテゴリを横断検索）と「＋ 新規登録」ボタン
- 上部サマリ: 合計金額と「申請中」「精算済」の件数を小さく表示
- 下部: 経費カードの縦リスト（作成日時の新しい順、スクロール可）
- 各カード: 日付／カテゴリ／金額（大）／ステータスバッジ／アクセントカラーの縦バー
- 選択中のカードはハイライトする
- カードをクリックで右ペインに詳細を表示

### 右ペイン（モード切替: 空 / 詳細 / フォーム）

**モード1: 空状態**
- 何も選んでいない初期状態
- 「左から経費を選んでください」のガイドを中央に表示

**モード2: 詳細**
- 上部: 右側に「編集」「削除」ボタン
- 中央: 金額（大見出し）・カテゴリ・日付・ステータスバッジ
- 下部: メモ・登録日時を行ラベル形式で表示
- ステータスの切替ボタン（「精算済にする」／「申請中に戻す」）を配置
- 削除時は確認ダイアログを出す

**モード3: 登録・編集フォーム**
- 上部: 「キャンセル」ボタン／タイトル（新規登録 or 編集）
- 中央: 各項目の入力フィールド（日付・カテゴリ・金額・メモ）
- 下部: 「保存」ボタン
- 保存後は詳細モードへ切り替わる
- 新規登録時のステータスは自動で「申請中」

## 操作フロー（CRUD）

左ペインは常に表示されている前提で、右ペインの状態が遷移する。

| 操作 | 右ペインの遷移 |
|---|---|
| 作成 (Create) | 空 or 詳細 → ＋新規登録 → フォーム → 保存 → 詳細 |
| 一覧 (Read) | 起動時に左ペイン表示。検索ボックスで絞り込み |
| 詳細 (Read) | カードクリック → 詳細 |
| 更新 (Update) | 詳細 → 編集 → フォーム（既存値入り）→ 保存 → 詳細 |
| 削除 (Delete) | 詳細 → 削除 → 確認 → 空状態 |
| ステータス切替 | 詳細 → 切替ボタン → 詳細（バッジと左ペインが即時更新） |

## 永続化

- `localStorage` のキー `expense-records` に配列として保存
- 起動時に読み込み、変更時に都度書き込む

## 初期データ

空だと触感がゼロなので、初回起動時にサンプル5件を投入する（localStorage未設定のときのみ）。
カテゴリと金額はバラけさせ、ステータスも「申請中」「精算済」を混在させる。

## 対象外（今回作らない）

- レシートOCR（画像からの自動入力）
- 承認ワークフロー（申請者／承認者の役割分担）
- クラウド保存・他端末共有（S6でSupabase連携するときに拡張）
- ログイン・マルチユーザー
- 会計ソフト連携・仕訳出力
- インポート／エクスポート
- CSV/PDF出力

## デザイン方針

- ツーカラムレイアウト（左=一覧リスト、右=詳細・フォーム）
- 白基調、アクセントはClaudeオレンジ `#c15f3c`
- 余白多め、角丸 `rounded-lg`、影は控えめ
- ステータスバッジで視覚的に状態を区別
- モバイル対応は今回スコープ外（PC表示を前提とする）
