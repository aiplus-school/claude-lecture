---

## 1. 作りたいアプリ

ひとり用の経費管理アプリ。紙のレシートや表計算ソフトで管理する代わりに、ブラウザ1枚で経費を登録・一覧・検索・編集・削除できる最小構成。業務用SaaSの核機能（OCRや承認ワークフローを除くCRUD）を再現する。

---

## 2. 技術制約

- HTML / JavaScript（vanilla、フレームワークなし）
- Tailwind CSS は CDN で読み込む
- データは localStorage に保存（サーバー・DB・外部API は使わない）
- ビルドツール・npm は使わない
- ファイルは index.html / app.js / styles.css の3枚で完結させる

---

## 3. 画面構成

### 3.1 全体レイアウト

ツーカラム構成。左ペインで一覧・検索・合計サマリ、右ペインで詳細・フォームを扱う。モバイル対応は対象外（PC表示のみ）。

### 3.2 左ペイン（幅 380px 固定）

- 上部に合計金額・申請中／精算済の件数を表示するサマリカード
- 検索ボックスと「＋ 新規登録」ボタン
- 下部に経費カードの縦リスト（作成日時の新しい順）
- 各カードに日付・金額・カテゴリ・メモ抜粋・ステータスバッジを表示
- 選択中のカードはハイライト表示

### 3.3 右ペイン（flex-1、モード切替）

以下の3モードを切り替えて表示する。

- 空状態（何も選んでいない初期表示）
- 詳細（選択中の経費を表示・ステータス切替ボタンを配置）
- 登録・編集フォーム

---

## 4. データ項目（経費1件）

| 項目 | キー | 必須 | 備考 |
|---|---|---|---|
| ID | id | ○ | 自動採番 |
| 日付 | date | ○ | 経費発生日（YYYY-MM-DD） |
| カテゴリ | category | ○ | 交通費／会議費／接待費／消耗品／通信費／その他 |
| 金額 | amount | ○ | 円（整数、0円不可） |
| メモ | memo |  | 用途や訪問先など自由記述 |
| ステータス | status | ○ | `pending`（申請中）/ `settled`（精算済）。初期値は `pending` |
| 作成日時 | createdAt | ○ | ISO文字列で自動付与 |

localStorage キー: `expense-records`（配列で保存）

---

## 5. CRUD操作フロー

- **Create**: 新規登録ボタン → フォーム → 保存 → 詳細表示（ステータスは自動で `pending`）
- **Read（一覧）**: 起動時に左ペインで一覧表示
- **Read（検索）**: 検索ボックスでメモ・カテゴリを横断検索
- **Update**: 詳細 → 編集ボタン → フォーム（既存値入り）→ 保存 → 詳細
- **Delete**: 詳細 → 削除ボタン → 確認ダイアログ → 空状態に戻る
- **ステータス切替**: 詳細 → 「精算済にする」／「申請中に戻す」ボタン → バッジとサマリが即時更新

---

## 6. デザイン方針

- 白基調、アクセントカラーは `#c15f3c`（Claudeオレンジ）
- ステータスバッジ: 申請中はアンバー系、精算済はグリーン系の淡いバッジ
- フォントは游ゴシック系（`"游ゴシック", "Yu Gothic", sans-serif`）
- 角丸は `rounded-lg`（8px）で統一
- 影は控えめ（`shadow-sm` 程度）
- 並び順は日付の新しい順

---

## 7. 初期データ

初回起動時（localStorage が空）のときだけ、サンプル5件を自動投入する。カテゴリと金額はバラけさせ、`pending` と `settled` を混在させる。

---

## 8. 対象外（今回作らない）

- レシートOCR（画像からの自動入力）
- 承認ワークフロー（申請者／承認者の役割分担）
- クラウド保存・他端末共有
- ログイン・マルチユーザー
- 会計ソフト連携・仕訳出力
- CSV/PDF出力
- モバイル対応
