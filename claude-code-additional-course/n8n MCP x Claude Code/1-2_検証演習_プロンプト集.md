## 案A：フォーム → スプレッドシート → Slack通知 を作る

### P1. 設計の相談（まだ作らせない）

```
n8nで業務自動化のワークフローを作りたいです。
Webの問い合わせフォームに入力が来たら、その内容をGoogleスプレッドシートに1行追加して、
Slackの #n8n-test チャンネルに「新しい問い合わせがありました」と通知するワークフローを作りたい。

まだ作らないでください。まず、
1. どのノードを使うか
2. それぞれのノードで何を設定する必要があるか
3. 事前に私が手作業で用意しておくべきもの（認証など）
を、n8n MCPで実際のノード仕様を確認したうえで教えてください。
```

### P2. ワークフローを生成

```
その構成でワークフローを作ってください。
- フォームの入力項目は「名前」「メールアドレス」「問い合わせ内容」の3つ
- スプレッドシートには「日時 / 名前 / メール / 問い合わせ内容」を記録
- Slack通知には送信者の名前と問い合わせ内容を含める
作成したら、validate_workflow で問題がないか検証してから提示してください。
```

### P3. n8nに反映（APIキーありの場合／自動反映）

```
検証が通ったら、このワークフローを私のn8nインスタンスに作成してください。
本番ではなくテスト用として、非アクティブ（オフ）の状態で作ってください。
```

### P4. 実行して詰まったときの修正依頼

```
実行したら〔ここに実際のエラーメッセージや症状を貼る〕という状態になりました。
原因を特定して、ワークフローを修正してください。
```

## 案C：壊れたワークフローを検証・修正させる

### P5. 検証（原因調査）

```
このワークフロー〔JSONを貼るか、n8n上のワークフローIDを指定〕を実行したら、
Slackには通知が来るのにスプレッドシートの名前欄が空のまま記録されます。
validate_workflow と、各ノードの設定を確認して、原因を調べてください。
```

### P6. 修正＋平易な説明

```
原因が分かったら修正して、修正前と修正後で何が変わったのかを、
n8nを知らない人にも分かるように説明してください。
```
