# 3-1. 概要・実用事例のリサーチ

## 1. Higgsfield とは何か

### 一言でいうと
**AIで画像・動画を生成するプラットフォーム。** 本来はブラウザ上のサービスとして使う。

複数の生成AIモデル（Sora 2、Veo 3.1、Kling 3.0 など）を**1つのサービスからまとめて使える**のが特徴。個別にAPIキーを取ったり契約したりせずに済む。

### Higgsfield MCP とは
その画像・動画生成機能を、**Claude Code などのAIエージェントから直接呼び出せるようにする接続口**。

- サーバーURL：**`https://mcp.higgsfield.ai/mcp`**
- **APIキー不要**。Higgsfield アカウントでログインするだけ
- 対応エージェント：Claude（web / デスクトップ / モバイル）、OpenClaw、Hermes Agent、NemoClaw など MCP対応クライアント全般

### 役割分担（講座で説明すると腹落ちする）
```
Claude Code  ←→  MCP  ←→  Higgsfield
   ↓                          ↓
やりたいことを解釈し、      実際の生成
自然言語を「生成の         （レンダリング、音声同期、
設定値」に翻訳する          キャラの一貫性維持）
```
- **Claude Code**：意図を解釈し、自然言語を構造化された生成パラメータに変換する
- **MCP**：両者をつなぐ安全な橋
- **Higgsfield**：実際の生成を担当し、完成物をチャットに返す

## 4. 🚨 料金・クレジット（3テーマ中もっとも慎重に扱う必要あり）

**ここを外すと受講者に実害が出る。** 生成のたびに実費（クレジット）が飛ぶ。

### プラン
情報源により表記が揺れているため**両方を記載する（要検証）**。
- **表記A**：Free / Starter / Plus / Ultra / Business の5プラン
- **表記B**：Basic 150クレジット / Pro 600クレジット / Ultimate 1,500クレジット

**無料プラン：10クレジット/月**。ウォーターマーク（透かし）あり。
**商用利用は有料プランから。** 透かしなしで管理するのが基本。
支払いは月払いと年間契約（年間は割引あり）。サインアップ時に無料クレジットが付く。

|---|---|---|

| **画像編集** | Product placement | 1クレジット（1回4枚） |

※ モデル・解像度・尺・音声有無で変わるため**固定ではない**。

### 🚨 講座として絶対に外せない事実
> **無料プランは10クレジット/月。5秒動画1本で約15クレジット。**
> **＝ 無料プランでは5秒の動画すら1本も作れない。**

受講者に「無料で試せます」と言ってはいけない。**課金が前提**であることを最初に明言する。

### 節約のコツ（実践知）
- **デフォルトは2K解像度・最高品質(High)。** 気になる場合は**1Kや「Medium」を指示**すると節約できる（note/さきすた）。
- Claudeとの会話自体はクレジットを消費しない（会話分のトークンのみ）。
- **バッチ処理前にクレジット残高を確認する。** 大量のVeo 3.1動画生成は計画的に（claudefa.st）。

> 🚨 **AIに「20パターン作って」と任せると、意図せずクレジットが溶ける。** これは講座で必ず警告する。

## 5. セットアップ（約1分）

1. Claude の **設定 > コネクタ** を開く
2. **「カスタムコネクタを追加」**を選択
3. 名前：`Higgsfield`、URL：`https://mcp.higgsfield.ai/mcp`
4. 「接続」→ Higgsfield アカウントでログイン
5. 許可内容（本人確認、プロフィール、メールアクセス）に同意
6. 権限を「常に許可」に設定（推奨）

**既存のHiggsfieldプランのクレジットがそのまま引き継がれる。** APIキーやOAuth設定は不要。

## 6. なぜ「Claude Code × Higgsfield MCP」なのか

### 🚨 本テーマ最大の勘所
> **「ブラウザで直接使えばいいのでは？」**

これに答えられないと動画が成立しない。受講者は必ずこう思う。

### 答え
**1枚作るだけならブラウザで十分。価値が出るのは「考えさせて・大量に・繰り返す」とき。**

| 価値 | 内容 | ブラウザだと |
|---|---|---|
| **① AIがプロンプトを考える** | 「話題の投稿を分析して、それに合う画像を20パターン」 | 人間が20個プロンプトを考えるのは無理 |
| **② 一度の指示で大量生成** | Claudeの指示による**自律的な複数パターン生成**（最大のメリットとされる） | 1枚ずつ手で回す |
| **③ 他の作業と繋がる** | Claude Codeで構築中のサイトに、必要な画像をその場で生成して埋め込む | 生成→DL→フォルダ→読込の**往復** |
| **④ 会話の文脈を保持** | 文脈を保ったまま生成、会話形式で修正・変更 | 毎回プロンプトを書き直す |
| **⑤ Claudeの能力とメモリを活用** | 分析→生成を一気通貫 | 分析は別ツール |

### ⚠️ 情報の限界（正直に記録）
- **正確なモデル一覧を公表している第三者ソースは存在しない。** 各社「selected models」としか書いていない。

### ✅ 解決（ほぼ）：無料プランは「10クレジット/月」でも「10/日」でもなく、**そもそも正式なFreeプランが存在しない**
前回「10/月 vs 10/日」で矛盾していた件を追跡した結果：

| ソース | 記述 |
|---|---|
| **imagine.art** | **「Higgsfieldは料金ページに無料プランを公開していない」**。ただし新規ユーザーは**「limited daily credits」**でアクセスできる |
| **Higgsfield公式ブログ** | 有料3プランのみを掲載。**「Freeプランは存在しない」** |
| Scopeful | 「無料枠はある。出力数・解像度・生成尺・商用利用権に上限」 |
| romptn | 「Free $0 / 10クレジット/月」← **旧プラン名を使っており情報が古い** |

**結論**：
- **「Freeプラン」という正式なプランは料金ページに存在しない。**
- 新規ユーザーに**少量の日次クレジット**が付く（＝「10/日」説の方が実態に近い）。**ただし正確な量は第三者情報では確定できず。**
- **どちらにせよ、無料枠は「透かしあり＋商用利用不可」**。→ **講座を無料枠前提で組むことはできない**、という実務上の結論は**変わらない**。

> ⚠️ **正確な無料枠の量は、実際にアカウントを作って確認するしかない。**（公式料金ページはJS描画のため自動取得不可、ブラウザからも読み取れず）

# 3-2. 「実際に使ってみる」検証アイデア 3案（提案）

> 選定基準：①「ブラウザで使えばいい」への答えになる ②クレジット消費を抑えられる ③収録尺に収まる

## 案A：1つの指示から画像を複数パターン一括生成 → 比較【AIに考えさせる】
- **内容**：「〇〇のサムネ案を5パターン作って」と頼み、Claude Code に**プロンプトを考えさせて**まとめて生成、並べて比較する。
- **狙い**：**「人間がプロンプトを考えなくていい」**というMCPならではの価値。ブラウザでは1枚ずつ手で書く作業になる。
- **Claude Codeの価値が出る点**：発想（プロンプト設計）と反復をAIが担当する。事例「サムネ5パターン」と完全一致。
- **クレジット試算**：**Soul（Basic）なら1クレジットで4枚**。かなり安い。**まずこれで検証すべき。**
- **想定リスク**：枚数をAI任せにすると消費が読めない。**「5枚」と明示し、モデルも指定**して収録する。

## 案B：画像を作る → その画像から動画化する【一連の流れ】
- **内容**：まず静止画を生成し、気に入ったものを**参考画像として動画を生成**する（最大15秒）。
- **狙い**：画像→動画の流れを1つの会話で完結させる、というエージェントらしさ。事例「UGC風画像20パターン→動画化」の縮小版。
- **Claude Codeの価値が出る点**：工程をまたいだ引き継ぎが不要。
- **クレジット試算**：🚨 **動画は高い。** 3〜5秒Liteで3〜5クレジット、Standardで7〜10。**8秒のVeo 3は70クレジット。**
  → **Lite / 3〜5秒 / 1本に絞る。Veo 3は収録で気軽に使わない。**
- **想定リスク**：待ち時間が長い（非同期・ポーリング）。**編集で待ち時間をカットする前提。**

## 案C：制作物に必要な素材をその場で生成して組み込む【他テーマとの連携】
- **内容**：Claude Code で簡単なLPやスライドを作りながら、必要になったヒーロー画像やアイコンを**その場で Higgsfield に生成させて埋め込む**。
- **狙い**：**「ブラウザで作って、ダウンロードして、フォルダに入れて、読み込んで…」という往復が丸ごと消える。** ここが「なぜMCPなのか」の最も強い答え。
- **Claude Codeの価値が出る点**：素材生成と制作物が同じ場所で完結する。**3テーマ目＝講座全体の締めにふさわしい。**
- **クレジット試算**：素材2〜3枚。Soulなら数クレジット。安い。
- **想定リスク**：作るもの（LP等）の準備が必要で工程が増える。

## 推奨の組み立て
**案A（AIに考えさせて量産）→ 案C（制作物にその場で組み込む）**を軸に、案Bは案Aの発展として**短く**挟む。

**理由**：
1. 本テーマは**「ブラウザで直接使えばいいのでは？」に答えられるかが勝負**。
2. **案Aが「量と発想」**、**案Cが「往復の消滅」**という2つの答えになる。
3. 案Bは絵として映えるが、**それ単体では「ブラウザでもできる」に負ける**。かつ最もクレジットを食う。
4. 案A・Cは画像中心＝**クレジット消費が軽い**（Soulなら1クレジットで4枚）。収録の実費が抑えられる。

### 🚨 収録前の必須確認
- クレジット残高と、1回の生成あたりの消費量を**実測**する
- 試算：案A（画像5枚）+ 案B（Lite動画1本）+ 案C（画像2〜3枚）で**何クレジット必要か**
- 撮り直しが発生する前提で、**必要量の2〜3倍の残高**を確保しておく
